## 📂 涉及的文件

### 新增文件

- **`src/router/modules/baseData.ts`** - 基础数据管理模块的静态路由配置

### 修改文件

- **`src/router/modules/remaining.ts`** - 引入并使用了 baseDataRouter

## 🎯 开发阶段使用

1. **直接访问菜单** - 所有页面都可以通过左侧菜单直接访问
2. **URL 访问** - 可以直接通过 URL 访问，例如：
   - `http://localhost/base-data/security-device/monitor-device`
   - `http://localhost/base-data/building-area`
3. **正常开发** - 就像正式配置的菜单一样进行开发

## 🔄 迁移到后端动态菜单

当准备对接后端时，按以下步骤操作：

### 步骤 1：在后端配置菜单

参考 `BASEDATA_MENU_CONFIG.md` 文档，在后端系统中配置菜单。

### 步骤 2：删除静态路由文件

```bash
# 删除静态路由配置文件
rm src/router/modules/baseData.ts
```

### 步骤 3：恢复 remaining.ts

在 `src/router/modules/remaining.ts` 中删除以下内容：

```typescript
// 删除这行导入
import baseDataRouter from './baseData'

// 删除数组中的这行
baseDataRouter,
```

### 步骤 4：测试验证

1. 重新登录系统
2. 确认菜单能正常显示
3. 测试各个页面能正常访问

## ⚠️ 注意事项

### 路由配置一致性

静态路由配置与后端配置保持了一致性，包括：
- ✅ 路由路径（path）
- ✅ 组件名称（name）
- ✅ 菜单标题（title）
- ✅ 菜单图标（icon）
- ✅ 层级结构

这样确保迁移时不会有任何问题。

### 权限控制

- 静态路由**没有权限控制**，所有用户都能看到
- 迁移到后端后，会自动应用权限控制
- 开发时注意：实际功能仍需实现权限验证

### 缓存配置

所有页面默认都启用了缓存（`noCache: false`），如果某些页面不需要缓存，可以在后端配置时调整。

## 🚀 快速命令

### 查看静态路由配置

```bash
cat src/router/modules/baseData.ts
```

### 验证路由是否生效

1. 打开浏览器控制台
2. 输入：`console.log($router.getRoutes())`
3. 查找 `base-data` 相关路由

### 临时禁用静态路由（不删除文件）

在 `src/router/modules/remaining.ts` 中注释掉：

```typescript
// baseDataRouter,  // 临时禁用
```

## 📞 遇到问题？

如果遇到以下问题：

### 菜单不显示

1. 确认文件路径正确
2. 检查浏览器控制台是否有错误
3. 清除浏览器缓存重试

### 页面 404

1. 确认 Vue 文件路径与路由配置一致
2. 检查文件是否存在
3. 查看组件导入路径是否正确

### 迁移后菜单不显示

1. 确认后端菜单已正确配置
2. 检查角色权限是否分配
3. 重新登录使权限生效
